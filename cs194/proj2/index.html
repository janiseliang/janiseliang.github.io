Image "Sharpening" (aka unsharp masking)

In this technique, we use a gaussian filter (size 20, sigma 4) to obtain the low-frequency part of the image. Then, we subtract the low frequencies from the original image and obtain only the high-frequency parts.

I experimented with adding different multiples (​alpha) of the high-frequency part to see if the image got any better. However, I only noticed higher color contrast and additional blocky artifacts for the higher multiples. An alpha of 0.5 seems to produce the most natural-looking result on the Taj Mahal image.

Since we are basically throwing away data when we blur an image, it is not possible to fully recover the original. Hybrid Images

Hybrid images can be created by combining the low frequencies of one image and the high frequencies of the other. We first must align the images by selecting two anchor points - in my images, I use the eyes of whatever creatures make up the hybrid. Similar to the previous part, I added a scalar multiple of the high frequencies (alpha) to help with adjusting the images.

Gaussian and Laplacian Stacks

For the Gaussian stack, we can simply convolve a Gaussian kernel repetitively on the image. Then, we create the Laplacian stack by computing the difference between each of the elements of the Gaussian stack. The final level of the Laplacian stack is equal to the last Gaussian element since there is nothing else to subtract.

The original image can be recovered by summing all of the Laplacian levels. We will use this when performing multiresolution blending. 

Above: Gaussian and Laplacian stacks for po-jack single (R) channel. The Laplacians have been min-max normalized in order to improve image contrast and visibility.

Above: Gaussian and Laplacian stacks for oraple.jpg (downloaded from class site).  The top row is the Gaussian stack, and the bottom is the Laplacian stack. I process each RGB channel separately, so the color images are created by combining the 3 channels for each level.

Multiresolution Blending

In order to perform multiresolution blending, we first create a mask to select a part of one image, and a different part of the other image. For the oraple, we can use a very simple mask with the left half of the apple and the right half of the orange selected.

apple

mask

orange

Then, we create a gaussian stack with the mask with the same length as the image stacks. For each level of the blending process starting at the blurriest level, we will apply a mask to the first image, and 1-mask to the second image. Then, we can sum all of the images together (both stacks and every level) for the final blended image.

This didn't work out so well because I had to downsize Wall-E a lot so that he wasn't a giant - but that made him look very pixelated. In addition, the background of the Wall-E image is very different than the background of the Kung Fu Panda image, so it looks like there's a glow around Wall-E. It seems pretty important that the background of the images match in color intensity, otherwise we end up blending colors in a weird way.

One thing I noticed when working with very high-definition images is that the convolutions start taking a very long time. The larger images require a larger gaussian kernel and they have more pixels to convolve in the first place - suggesting a runtime like O(N^3), which is very slow. It seems that the pyramid method would be more efficient, but since I already implemented stacks, I didn't really want to go back to redo it.

Unfortunately the black background of the blood moon image gets "leaked" into the blended image. This is undesired but also seems unavoidable with this method, unless we start with two images that have similar backgrounds. The effect is quite interesting though, since the background behind the moon looks somewhat like a shadow special effect (applied to the image itself for comparison).
